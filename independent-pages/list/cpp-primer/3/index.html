第 3 章 字符串、向量和数组

命名空间的 using 声明

  • 每个名字都需要独立的 using 声明；

  • 头文件中不应包含 using 声明（否则每个使用了这个头文件的文件都会含有这个声明，可能会产生名字冲突）；

标准库类型 string（可变长的字符序列）

string 对象的初始化

  • 拷贝初始化 和 直接初始化 （是否使用了 “=”）。

string 对象上的操作

读写

  • cin / cout 读写忽略空格；

  • 读取未知数量的 string 对象；

  • getline 读取整行（直到读入换行符为止，但不存换行符）；

  • 触发 getline 函数返回的换行符被丢弃掉了，得到的 string 对象中不包含该换行符。

empty 和 size

  • empty ；

  • size ；

string::size_type 类型

  • 配套类型体现了标准库类型与机器无关的特性；
  • 允许编译器通过 auto 或 decltype 来推断变量的类型（C++ 11）；

比较与相加

  • string 大小与字符有关；
  • 字面值不能相加，string 对象与字面值可以；
  • 字符串字面值与 string 是不同的类型。

处理 string 对象中的字符

  • cctype 头文件 中的函数；
  • 建议使用 C++ 版本的 C 标准库头文件（如 <cstring>）。

基于范围的 for 语句

下标

  • 下标运算符（[ ]） 接收的输入参数是 string::size_type 类型的值；
  • 使用下标执行迭代

标准库类型 vector（给定类型对象的可变长序列）

  • 头部信息；

  • 编译器根据模版创建类或函数的过程称为 实例化 ；

  • vector 是模版而非类型，应包含元素类型，如 vector<int> ；

  • vector 中元素还是 vector 的新旧标准差异（C++ 11）。

定义和初始化 vector 对象

  • 直接初始化和间接初始化；

  • 可以定义一个空的 vector ，在运行时获取到元素的值后再逐一增加；

  • 列表初始化是值初始化，直接初始化是元素数量和初始值；

  • 列表初始化；

  • 创建指定数量的元素；

  • 值初始化支持默认初始化；

  • 如果只提供了元素数量，没有设定初值，只能使用直接初始化。

添加元素

  • push_back ；

  • vector 对象能高效增长，提供 方法 进一步提升动态添加元素的性能；

  • 添加元素不得使用范围 for 循环。

其他操作

  • size 返回值类型；

  • 大小与元素值大小有关（前提是可以进行比较）；

  • 计算对象的索引的例子；

  • 不能使用下标形式添加元素；

  • 通过下标访问不存在的元素会造成 缓冲区溢出（buffer overflow），是导致应用程序出现安全问题的一个重要原因。

迭代器

使用迭代器

  • 获取迭代器不是使用 & ；

  • 如果容器为空，则 begin 和 end 返回的是同一个迭代器，都是尾后迭代器；

  • 所有标准库容器的迭代器都定义了 “ == ” 和 “ != ” ，养成使用它们的习惯；

  • 迭代器的类型；

  • begin 和 end 的返回值由对象是否是常量决定；

  • cbegin 和 cend 返回只读迭代器，忽略对象本身是否是常量（C++ 11）；

  • 箭头运算符（ -> ）；

  • 任何一种可能改变 vector 对象容量的操作都会使该 vector 对象的迭代器失效（如 push_back ）。

迭代器运算

  • 最接近中间元素的迭代器；

  • 带符号整数 difference_type ，两迭代器之间的距离；

  • 二分查找。

数组（大小固定）

定义和初始化

  • 数组纬度必须是一个常量表达式；

  • 默认初始化；

  • 不允许使用 auto 关键字由初始值的列表推断类型（auto 的到的类型是指针）；

  • 忽略数组纬度，显式初始化；

  • 字符串字面值初始化；

  • 不允许拷贝和赋值（一些编译器支持数组的赋值，是所谓的 编译器扩展 ，尽量避免使用）；

  • 复杂数组声明。

访问数组元素

  • 通常类型为 size_t ，是一种机器相关的无符号类型，定义在 cstddef 头文件 中；
  • 推荐使用范围 for 进行遍历；
  • 缓冲区溢出：下标越界，试图访问非法内存区域。

指针和数组

  • 大多数表达式中，使用数组类型的对象其实是使用一个指向该数组首元素的指针；

  • 可以使用 decltype 推断类型；

  • 指针也是迭代器；

  • 标准库函数 begin 和 end （非成员函数，定义在 iterator 头文件 中）

  • 尾后指针不能执行解引用和递增操作；

  • 指针运算问题；

  • 指针相减得到类型 ptrdiff_t ，同样定义在 cstddef 头文件 中，是机器相关的带符号类型；

  • 指向同一对象的指针可以进行比较；

  • 内置的下标运算符所用的索引值不是无符号类型，这一点与 vector 和 string 不一样。

C 风格字符串

字符串放在字符数组中以空字符（'\0'）结束。

cstring 头文件 中，C 风格字符串的函数：

  • strlen 函数的一个误区（正解：直到遇到空字符才会停止）；

  • 运算符作用在 C 风格字符串上，实际比较的是指针而非字符串本身，需要使用 strcmp ；

  • 对大多数应用来说，使用 标准库 string 要比使用 C 风格字符串更安全、更高效。

与旧代码的接口

混用 string 对象和 C 风格字符串

  • 允许使用以空字符结束的字符数组来初始化 string 对象或为 string 对象赋值；

  • 允许使用以空字符结束的字符数组作为 string 对象加法运算的一个运算对象（不能两个运算对象都是）；

  • string 对象的复合赋值允许使用以空字符结束的字符数组作为右侧的运算对象；

  • s_str 成员函数；

  • ⚠️ 如果执行完 c_str 函数后程序想一直都能使用其返回的数组，最好重新拷贝一份。

使用数组初始化 vector 对象

  • 指明拷贝区域；

  • 也可以是数组的一部分（传入地址）；

  • 📒 尽量使用 vector 和迭代器而非内置数组和指针；尽量使用 string 而非 C 风格的基于数组的字符串。

多维数组

实际上是数组的数组。

  • 初始化；

  • 下标引用；

  • ⚠️ 范围 for 遍历的一个重点：除了最内层循环外，其他控制变量都应是引用类型；

  • 指针和多维数组，圆括号不可少；

  • 使用 auto 或 decltype 避免在数组前面加上指针类型（C++ 11）；

  • 类型别名简化多维数组的指针。
